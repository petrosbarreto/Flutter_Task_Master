## 🎯 Como Usar Este Projeto

### ✅ Primeira Vez?

1. **Instale as dependências**:

   ```bash
   flutter pub get
   ```

2. **Configure o Supabase** (IMPORTANTE):
   - Siga o guia: `docs/SUPABASE_SETUP.md`
   - Atualize: `lib/config/supabase_config.dart`

3. **Teste no emulador**:
   ```bash
   flutter run
   ```

### 🔄 Workflow das Aulas

**Cada aula é uma branch cumulativa:**

1. **Ler o material** da aula (em `docs/`)
2. **Criar a branch** da próxima aula:

   ```bash
   git checkout aula/01-setup-inicial
   git pull origin aula/01-setup-inicial
   git checkout -b aula/02-layouts-estaticos
   ```

3. **Implementar a feature**:
   - Fazer commits pequenos e descritivos
   - Testar `flutter run`
   - Analisar `flutter analyze`
   - Formatar `flutter format .`

4. **Push da branch**:
   ```bash
   git push -u origin aula/02-layouts-estaticos
   ```

### 🗄️ Estrutura de Pastas

```
lib/
├── config/            # Configurações (Supabase, etc)
├── core/
│   ├── services/      # Serviços (Supabase, HTTP)
│   ├── constants/     # Constantes
│   └── utils/         # Utilitários
├── features/
│   ├── tasks/         # Feature de Tarefas
│   └── auth/          # Feature de Auth
└── main.dart
```

### ⚠️ Importante

- **Nunca commit chaves do Supabase!** Sempre `.env` ou variáveis de ambiente
- **Commits pequenos**: Um conceito por commit
- **Teste sempre**: `flutter run` antes de fazer push
- **Mensagens claras**: `feat:`, `fix:`, `chore:`, `docs:`

### 🆘 Problemas?

1. **Build não compila?**

   ```bash
   flutter clean
   flutter pub get
   flutter run
   ```

2. **Erro de Supabase?**
   - Verificar chaves em `lib/config/supabase_config.dart`
   - Conferir se o projeto existe em supabase.com

3. **Conflito de git?**
   - Veja: `docs/BRANCHES_WORKFLOW.md` → Troubleshooting
